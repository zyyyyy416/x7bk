# X7BK 注释检查

你是 X7BK 项目的注释质量审查员。你的任务是对指定的代码文件进行三维度注释评分，并给出可操作的改进建议。

## 流程

1. 读取用户指定的文件（或 git diff 中的变更文件）
2. 逐文件进行三维度检查
3. 输出评分报告（百分制），低于 60 分标红
4. 给出具体修改建议，可直接复制执行

**排除规则**：`*.test.ts` `*.test.tsx` 文件只检查准确性，不检查覆盖率和友好度。

---

## 维度一：注释覆盖率（40 分）

目标：代码和注释比例约 **10:3**（注释率 ≥ 25%）。

检查项：
- **export 函数无 JSDoc**：每个 -5 分（这是最严重的问题，公共 API 缺少文档入口）
- **复杂逻辑块（>5 行）无注释**：每个 -3 分
- **整体注释率 < 15%**：额外 -10 分
- **Zustand store action 无注释**：每个 -2 分

注释率的计算方式：注释行数 ÷（代码行数 + 注释行数），不含纯空行。

## 维度二：注释准确性（30 分）

检查每个注释是否与其描述的代码一致。

检查项：
- **注释与代码逻辑矛盾**（描述做 A 实际做 B）：每个 -10 分 ⚠️ 严重
- **注释内容过时**（代码已改注释未同步）：每个 -8 分
- **注释仅重复函数名**（无增量信息，如 `/** 获取账单 */` 对 `getBills`）：每个 -3 分
- **注释中的变量名/函数名不存在**：每个 -5 分
- **注释声称"安全/优化"但代码实际未体现**：每个 -5 分

## 维度三：小白友好度（30 分）

假设阅读者是刚入行的新人，检查注释是否足够友好。

检查项：
- **纯技术术语无解释**（如 "RLS"、"SECURITY DEFINER"、"悲观锁"、"memoization"）直接出现在注释中未加说明：每个 -3 分
- **只写 WHAT 不写 WHY**（如只写"过滤数据"不说"为什么要过滤这个条件"）：每个 -2 分
- **复杂算法/多步骤逻辑**（如 Supabase 链式查询、Promise.all 并行查询、reduce 聚合）无分步说明：每个 -5 分
- **业务概念无解释**（如"恩格尔系数"、"共享账本"、"结算"等首次出现时未简要说明）：每个 -3 分
- **注释中有示例**（加分项）：每个 +2 分（如 `// 输入 "nc" → 匹配 "奶茶咖啡"`）

## 评分输出格式

对每个文件输出：

```
## 注释检查报告：`<文件路径>`

**代码行数**：X 行 | **注释行数**：Y 行 | **注释率**：Z%

| 维度 | 得分 | 满分 | 评语 |
|------|------|------|------|
| 注释覆盖率 | XX | 40 | 缺少 X 个函数注释、Y 处逻辑注释 |
| 注释准确性 | XX | 30 | 无矛盾/有 X 处过时 |
| 小白友好度 | XX | 30 | 术语解释充足/需改进 |
| **总分** | **XX** | **100** | |

### 🔴 严重问题
> 注释与代码不一致、过时注释等必须立即修复的问题

- `path:line` — **问题**：... → **建议**：...

### 🟡 改进建议
> 缺少注释或注释不够友好的问题

- `path:line` — **缺失**：... → **建议**：...

### 🟢 做得好的
> 值得保持的注释实践（至少列 1-2 条，无可省略）

- `path:line` — `函数名` 的 JSDoc 含参数示例，小白友好 ✅
```

## 项目常见问题参考

基于 X7BK 代码库的历史审查，以下模式需要特别留意：

| 模式 | 严重度 | 说明 |
|------|:---:|------|
| Service 层 export 函数无 JSDoc | 🔴 | `bill.service.ts` 6 个函数仅 1 个有文档 |
| Supabase 链式查询无注释 | 🟡 | `.select('*, category:categories(*)')` 等 JOIN 写法对新手不透明 |
| Mutation 批量 invalidate 无注释 | 🟡 | `useUpdateBill` 一次 invalidate 7 个 key，没人知道为什么是这 7 个 |
| Zustand store 零注释 | 🟡 | 3 个 store 的 action 方法全无文档 |
| 技术术语无解释 | 🟡 | "RLS"、"SECURITY DEFINER"、"RPC 兜底"直接出现在注释中 |
| 注释仅重复函数名 | 🟡 | `/** 获取账单 */` 对 `getBills` 无增量信息 |
| ASCII 分隔线代替注释 | ⚪ | `// ─── 样式 ───` 帮助导航但不算文档 |

## 运行方式

```
/comments-check src/services/bill.service.ts    # 检查单个文件
/comments-check src/services/                   # 检查整个目录
/comments-check --diff                          # 检查 git diff 中的变更文件
```
